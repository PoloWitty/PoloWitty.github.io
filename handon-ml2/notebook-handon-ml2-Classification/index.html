one way to compare classifier.

Multiclass Classification

Some algorithms (such as SGD classifiers, Random Forest classifiers, and naive Bayes classifiers) are capable of handling multiple classes natively. Others (such as Logistic Regression or SVM classifiers) are strictly binary classifiers.

There are typically two ways to perform multiclass classification with multiple binary classifiers.

OvR and OvO strategy

OvR

one-versus-the-rest (also called one-versus-all)

train $N$ classifiers ($N$ is the number of classes), one for each class. When you want to classify a class, you get the decision score from each classifier and select the class whose classifier outputs the highest score.

OvO

one-versus-one

train a binary classifier for every pair of classes (for example, in this digits classifier problem, it can be one to distinguish 0s and 1s, another to distinguish 0s and 2s……)

If there are $N$ classes, you need to train $N * (N-1)/2$ classifiers.

The main advantage of OvO is that each classifier only needs to be trained on the part of the training set for the two classes that it must distinguish.

the choice between OvR and OvO

Some algorithm (such as SVM) scale poorly with the size of the training set. So it’s more faster to train many classifiers on small training sets than to train few classifiers on large training sets. So OvO is preferred for those algorighm. Otherwise, OvR is preferred.

Scikit-learn will automatically run OvR or OvO depending on the algorithm.
